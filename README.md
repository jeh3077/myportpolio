# Day1

#### Chapter 1. 컴퓨터 구조
 컴퓨터가 이해하는 정보, 컴퓨터의 네 가지 핵심 부품

1. 컴퓨터가 이해하는 정보
- 데이터와 명령어로 이루어져 있다.

  데이터 : 컴퓨터가 이해하는 정적인 정보
  
  명령어 : 데이터를 움직이고 컴퓨터를 작동시키는 정보
 - ex) 1과 2를 출력하라.
  데이터  :1, 2 / 명령어 : 출력하라

2. 컴퓨터의 4가지 핵심 부품
- CPU, 메모리, 보조기억장치, 입출력장치
  1. CPU : 산술논리연산장치, 제어장치, 레지스터 등으로 이루어져 있으며 컴퓨터의 두뇌라고 봄.
  2. 메모리 : 현재 실행되는 프로그램의 명령어와 데이터를 저장
  3. 보조기억장치 : 메모리의 약점을 보완함. 가격이 저렴, 전원이 꺼져도 내용 저장
  4. 입출력장치 : 마이크, 스피커, 프린터, 마우스, 키보드 등.

- 확인문제
1) 컴퓨터가 이해하는 정보에는 데이터와 명령어가 있습니다.
2) 명령어를 해석하고 실행하는 장치 -> CPU
3) 프로그램이 실행되려면 반드시 메모리에 저장되어 있어야 합니다.
4) 보조기억장치 : 보관할 프로그램 저장 / 메모리 : 실행되는 프로그램 저장
5) 시스템 버스에 관련한 내용 

#### Charter 2. 데이터
2-1. 0과 1로 숫자를 표현하는 방법 : 이진법
- 0과 1을 나타내는 가장 작은 정보 단위를 비트라고 한다.
- ex) 2비트로 표현할 수 있는 정보
  0 0 / 0 1 / 1 0 / 1 1
- 따라서 n개의 비트로 표현 할 수 있는 상태는 2의 n승가지
- 비트는 가장 작은 단위이므로 프로그램의 크기를 말할 때는 비트보다 큰 단위인 바이트를 사용한다.
- 바이트 : 여덟 개의 비트를 묶은 단위. (1 byte = 8 bit, 2의 8승 개의 정보를 표현)
- 이진법으로 표현하기
- 0과 1로 음수 표현하기 : 2의 보수

- 확인문제
  1) 2000MB -> 2GB
  2) 1000GB = TB / 1000kB = 1MB / 1000MB = 1GB / 1024 bit 

2-2. 0과 1로 문자를 표현하는 방법
- 문자 집합 : 컴퓨터가 이해할 수 있는 문자의 모음
- 인코딩 : 코드화하는 과정, 문자를 0과 1로 이루어진 문자코드로 변환
- 디코딩 : 코드를 해석하는 과정, 0과 1로 표현된 문자 코드를 문자로 변환
- 아스키 코드 : 7비트로 하나의 문자 표현(총 8비트인데 1비트는 오류검출을 위한 패리티 비트)
- EUC-KR : 문자 집합을 기반으로 하는 대표적인 완성형 인코딩 방식
- 유니코드 : 통일된 문자 집합, 한글, 영어, 화살표와 같은 특수 문자, 심지어 이모티콘까지
- 유니코드의 인코딩 방식 : utf-8, utf-16 등..

#### Charter 3. 명령어
3-1. 소스 코드와 명령어
- 고급 언어와 저급 언어
  1) 고급 언어 : 사람을 위한 언어
  2) 저급 언어 : 기계어, 어셈블리어
  ex) 기계어 -> 어셈블리어
     0101 0101 -> push rbp
     0101 1101 -> pop rbp
     1100 0011 -> ret
 - 컴파일 언어와 인터프리터 언어
   * 컴파일 ? 컴파일 언어로 작성된 소스 코드가 저급 언어로 변환되는 과정
   1) 컴파일 언어 : 컴파일러를 통해 저급 언어로 변환된 언어로 한꺼번에 실행
   2) 인터프리터 언어 : 인터프리터에 의해 소스 코드가 한 줄씩 실행되는 고급 언어, 대화형 가능
 - C 언어 컴파일 과정
   test.c -> [전처리기] -> test.i -> [컴파일러] -> test.s -> [어셈블러] -> test.0 -> [링커] -> test.exe

- 확인문제
  1) 고급 언어가 아닌 것은? 기계어,어셈블리어
  2) 1.컴파일 언어는 한 줄이라도 소스 코드상에 오류가 있다면 실행x
     2.일반적으로 컴파일 언어보다 인터프리터 언어가 빠르다-> 인터프리터 언어가 더 느리다
     3.인터프리터는 인터프리터 언어로 작성된 소스 코드를 한 줄씩 저급 언어로 변환하여 실행
     4.컴파일러는 컴파일언어로 작성된 소스 코드 전체를 목적 코드로 변환
